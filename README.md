# 🚨 AI-Powered Anomaly Detection for Fraudulent System Activities

## 📌 Project Overview

This project focuses on detecting anomalies in network traffic using
unsupervised machine learning techniques. The goal is to identify
unusual patterns that may indicate potential cybersecurity threats.

Based on real-world IT experience, traditional rule-based monitoring
systems are limited because they rely on predefined rules and cannot
easily detect new or unknown attack patterns.

## 🎯 Problem Statement

Detect anomalies in network traffic without relying on labeled data to
identify potential cybersecurity threats.

## 📊 Dataset

-   Records: 25,192\
-   Features: 42\
-   Includes traffic metrics, connection features, login indicators, and
    network attributes.

## 🧠 Models Used

-   Isolation Forest (Final Model)
-   K-Means
-   DBSCAN

## 🚀 Key Highlights

-   Unsupervised anomaly detection
-   Feature engineering with ratio-based features
-   Feature selection using VarianceThreshold
-   Bias auditing using subgroup analysis

## 📈 Results Summary

Isolation Forest performed best due to consistency and ability to handle
high-dimensional data.

## ⚖️ Limitations

-   No ground truth labels
-   Possible false positives/negatives
-   Model sensitivity
